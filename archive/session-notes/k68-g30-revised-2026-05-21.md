## Stage 1 — XCTest invariant (MANDATORY · load-bearing)

**Land target**: `cockpit/Tests/DemiurgeCoreTests/AbsorbedNeedsMeasured
OracleTests.swift` (new file) — 3 test method:

### Test 1 — `testAbsorbedRequiresMeasuredOraclePASS`

Iterate over a fixture set of `<X>VerifyRecord` instances (NOT
`exports/` scan — see Open Q #2). 각 record 에 대해:
- `absorbed == true` 이면 (a) `measuredOracle != nil` AND
  `measuredOracle.isMeasuredOraclePASS == true`, OR (b) record 가
  illustrative-physics cell (현 demiurge 에서 single occupant =
  `FusionVerifyRecord` 의 `mc_slab_demo` path).
- 둘 다 아니면 FAIL.

### Test 2 — `testAbsorbedNotAutoflippedByD95Computed`

Synth `EnergyVerifyRecord` with:
- `hexaNativeParity.parityStatus = "21/21 PASS at rel_err <=1e-13"`
  (so `isHexaNativeAbsorbed` computed → true)
- `measuredOracle = nil`
- `absorbed = true` (← 위반)

Assert: 본 record 가 Test 1 의 invariant 에 의해 FAIL 분류되어야 함
(stored `absorbed` 가 substrate-parity 의 D95 computed 부산물로 set
되는 path 를 차단 · D103 separation).

### Test 3 — `testD106IllustrativeCellExemptFromMeasuredOracle`

Synth `FusionVerifyRecord` with:
- `measuredOracle = nil`
- `absorbed = false`
- (cell 의 illustrative-physics gate 적용 — record 의 gate type
  field 또는 SkippedCellsAggregator heuristic)

Assert: 본 record 는 Test 1 의 invariant 에 의해 PASS 분류 (D106
exclusion clause — anti-conflation · RFC 013 §6.12).

### G29 dependency

Test 1 의 fixture set 은 G29 가 첫 cell `absorbed=true` 를 *measured-
oracle PASS* 근거로 land 한 *후* 에야 production fixture (Energy/
solar의 real record) 가 들어옴. G29 land 전에 Stage 1 을 land 하면
fixture 는 synth (Test 2/3) 중심 · Test 1 의 real-data branch 는
G29 land 시 첫 검증.

**land sequencing**: G29 land 시 Stage 1 fixture 가 real data 위에
build 가능. G29 land 전 Stage 1 만 단독 land 도 가능 (synth fixture
만 · invariant 자체는 검증 · 단지 real-data path 가 *aspirational*
한 상태).

## Stage 2 — `.specify/memory/constitution.md` row (DEFERRED)

`.specify/memory/constitution.md` 가 현재 line 1 `# [PROJECT_NAME]
Constitution` template-only. 본 file 의 populate 가 별도 사용자
결정 (κ-68 G30 scope 밖). populate 후 add row 후보:

```markdown
### Cell absorbed = measured oracle gate (D109 · κ-68 RFC 013 §6.11)

**Rule**: stored `<Record>.absorbed: Bool = true` requires a non-nil
`measuredOracle: MeasuredOracleRef?` field with
`measuredOracle.isMeasuredOraclePASS == true`. D106 illustrative-
physics cells (Fusion mc_transport · pattern-proof kernels) are
exempt — they MUST keep `measured_oracle = nil` AND
`absorbed = false`.

**Why**: substrate-parity ≠ measurement-parity (D103 dimension
separation). D95 computed `isHexaNativeAbsorbed` projects substrate
parity; stored `absorbed` carries measured-oracle truth. Conflation
silently flips the latter from the former — exactly what RFC 013
§6.11 + §6.12 anti-conflation guard against.

**Enforcement**: `cockpit/Tests/DemiurgeCoreTests/AbsorbedNeedsMeasured
OracleTests.swift` (XCTest invariant · Stage 1) — load-bearing.
This constitution row is doc-shaped audit (the runtime guard lives
in test).

**Source decisions**: D103 (dimension-separation docstring) · D106
(illustrative-physics gate · GateType case) · D109 (Energy/solar
cell pick · κ-68 G27 land).
```
