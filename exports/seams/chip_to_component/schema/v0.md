## §A — Schema (JSON rendering of the HXC key set)

Wire form = HXC v2 canonicalisation of the same key set (A1–A35
deterministic). JSON for human read. Units explicit. `null` =
"chip HANDOFF did not supply" — never a fabricated default (g3).

```jsonc
{
  "interface": "hexa-arch:seam:chip->component:handoff-dossier",
  "schema_version": "0.1",
  "record_id": "<content-hash of body, HXC A12-canon hex>",
  "produced_at_utc": "<ISO-8601 UTC — set by hexa-arch[chip] HANDOFF>",

  "die": {
    "node":          "<process node string>",
    "area_um2":      "<number|null>",
    "pad_count":     "<int|null>",
```

## §B — Consumer rules (hexa-arch[component] SPECIFY)

- A record **without** a `provenance` block is **rejected fail-loud**
  (no silent default — g3 / `AGENTS.tape @F f4`).
- `provenance.measurement_gate` reflects the chip pipeline's **real
  state** (today: NoC `GATE_B_PINNED_MET`, synth/P&R design-only);
  hexa-arch renders it on every consuming output (D16) and **never
  upgrades** it.
- Unknown extra keys ignored (forward-compatible, rfc_002 §6 idiom).

## §C — Versioning

v0.x = draft. → **v1.0 only when** a real chip HANDOFF dossier is
emitted by a *measured* chip pipeline pass and validates (rfc_008
§6). No `absorbed`/`resolved`/"wired" claim before that (g3).
