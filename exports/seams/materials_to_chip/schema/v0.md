## §B — Consumer rules (hexa-arch[chip] SPECIFY)

- A record **without** a `provenance` block is **rejected fail-loud**
  (no silent default — g3 / `AGENTS.tape @F f4`).
- hexa-arch **renders** `provenance.measurement_gate` / `absorbed` on
  every consuming output (D16 cockpit honesty feature) but **never
  asserts or upgrades** them — carried verbatim from upstream.
- Unknown extra keys are ignored (forward-compatible, rfc_002 §6
  idiom). chip SPECIFY consumes only the subset its active device/PDK
  model declares.

v0.x = draft, upstream HANDOFF shape unpinned. → **v1.0 only when**
