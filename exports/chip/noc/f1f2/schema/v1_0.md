(algorithms A1–A35 deterministic). JSON is shown here for human read.

## §C — Provenance enforcement summary

`provenance.absorbed = false` is the **default** until rfc_001 §8
measurement gate closes (`GATE_CLOSED_MEASURED`). Records emitted before
that point are capture-only, not "absorbed at measured parity" claims.
A run that violates the Leighton oracle exits 91 (rfc_001 §7.3) and the
record MUST NOT be emitted.

See `proposals/rfc_002_f1f2_export_interface.md` §4 for the full field
table and enforcement matrix.

## §D — Pair-record convention

A single record describes one `(topology, traffic, node)` run. F1/F2
verdict is derived by the consumer from **two records** sharing identical
`{ traffic, node, schema_version, wire_delay_model.node, wire_delay_
model.ps_per_mm }` but differing in `topology.degree`. The pair verdict
is written by an aggregator as a derived record with `interface =
"hexa-arch:chip:noc:F1F2-pair-verdict"` to `../pair_verdicts/<pair_id>.
hxc`; same provenance discipline.
