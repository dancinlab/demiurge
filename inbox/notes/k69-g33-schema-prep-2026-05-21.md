## § 1. `MeasuredOracleRef` schema reuse audit (8-field shape)

SSOT: `cockpit/Sources/DemiurgeCore/Models/MeasuredOracleRef.swift` (107 lines
· κ-68 G27 land @ `4a1a087`).

| # | field                | type      | required | Energy-specific? | G33-readiness verdict |
|---|----------------------|-----------|----------|------------------|-----------------------|
| 1 | `oracleSource`       | String    | yes      | NO — header doc cites NREL example but the type is just a citable string | clean reuse — Aura: `"PhysioNet Sleep-EDF Expanded · SC4001E0 · Fpz-Cz · 30-s Wake epoch · 100 Hz"` |
| 2 | `unit`               | String    | yes      | NO — header doc *mentions* W/m² but no semantic constraint | clean reuse — Aura: `"V^2"` (band-integrated PSD) or `"dB"` (alpha-peak ratio) |
| 3 | `sampleCount`        | Int       | yes      | NO | clean reuse — Aura: # epochs × # channels or # spectral bins, depending on PASS criterion |
| 4 | `meanRelErr`         | Double    | yes      | NO — `isMeasuredOraclePASS` is `meanRelErr <= threshold` generic | clean reuse |
| 5 | `maxRelErr`          | Double?   | optional | NO | clean reuse — nil if Aura producer reports summary stats only |
| 6 | `threshold`          | Double    | yes      | NO | clean reuse — `0.05` (mirror G29 5% solar criterion) per G32 candidate-A default (a) |
| 7 | `datasetCaveats`     | String    | yes (D80 honesty floor) | NO | clean reuse — Aura: clear caveat re. single-subject, montage, line-noise filter |
| 8 | `datasetCitation`    | String?   | optional | NO | clean reuse — Aura: `"https://physionet.org/content/sleep-edfx/1.0.0/"` |

**Verdict**: schema is **fully generic by construction**. The κ-68 G28 type
header (`MeasuredOracleRef.swift` lines 1–25) calls out Energy as an *example*
("e.g. EnergyVerifyRecord"); no Energy-specific field, no Energy-specific
constraint, no Energy-keyed JSON convention. **Zero Energy-isms to relax.**

Subtle G28→G33 generalization signal worth flagging: the unit examples in the
field doc-comment are physical (`W/m^2`, `K`, `1`). Aura signal-proc units
(`V^2`, `dB`) are equally valid but the comment doesn't enumerate
signal/spectral units — purely doc-cosmetic, NOT a code change. Could be a
one-line doc-comment refresh on G33 land if desired (not required).

## § 4. Producer path (hexa-lang side)

Candidate producer file: `~/core/hexa-lang/stdlib/aura/aura_mne.py`
(282 lines).

**Current emit shape**: Writes `aura_eeg_v1.meta.json` + `<fif>` artifact
+ stderr `AURA_MNE_RESULT <json>` summary. **Does NOT emit a verify-shaped
record** — it emits the `aura + analyze` artifact consumed by the cockpit
`AuraAnalyzeProducer.swift` and typed as `AuraRecord` (analyze record, NOT
verify). `absorbed=false ALWAYS` is hard-coded in the cockpit producer
(`AuraAnalyzeProducer.swift` line 266 · `provenance: AuraProvenance(absorbed:
false, ...)`) and in the Python comments (lines 41–42, 51–52).

**What G33 needs (G31 mirror)**:

The κ-68 G29 pattern (template) is `nrel_midc_pyranometer.py` (418 lines)
which:
1. Computes the measured-oracle stats via `_build_measured_oracle(date)`
   (lines 232–301) — fetches real data, applies filters, computes
   `mean_rel_err` / `max_rel_err`, builds the 8-field `measured_oracle` dict.
2. In `main()` (lines 314–414) emits a full **verify-shaped JSON record**
   matching `EnergyVerifyRecord` Codable shape — keyed by `"domain": "energy",
   "verb": "verify", ...`, with `absorbed: pass_flag` (legitimate flip) and
   `measured_oracle: measured_oracle` block populated.

**G33 producer work** = a new sibling to `aura_mne.py`:
- New file (recommended) — `~/core/hexa-lang/stdlib/aura/aura_physionet_eeg_verify.py`
  (mirror of `nrel_midc_pyranometer.py` for the Aura verify path), OR
- A second `main()` entrypoint within `aura_mne.py` (less clean — analyze
  and verify have different output dirs + different record schemas).

The new producer fetches the PhysioNet Sleep-EDF subject epoch via
`wget`/`urllib` (anonymous · CC-BY), pipes the signal through MNE's reader
+ Welch PSD, computes alpha-band power, compares against an oracle anchor
(e.g. published Sleep-EDF reference statistics or a single-subject expected
band ratio from the Sleep-EDF Expanded README), produces `mean_rel_err` over
N=5 Wake epochs, emits the `measured_oracle` block + the verify record JSON.

**Producer scope estimate**: ~250–350 LOC for `aura_physionet_eeg_verify.py`
(mirror of `nrel_midc_pyranometer.py` — same scaffolding: stamp + record-
dict serialization + summary stderr marker · oracle compute is comparable
complexity to MIDC fetch + clear-sky filter).

**Producer is the dominant G33 cost** — substantially larger than the §2
cockpit-Swift diff.

**Cockpit dispatcher side**: needs an `AuraVerifyProducer` analog of
`AuraAnalyzeProducer.swift` (~150 LOC mirror) that spawns the new Python
producer, decodes the verify JSON into `AuraVerifyRecord`, and persists
under `exports/aura/verify/<stamp>/`. (Or — if dispatcher already auto-
discovers verify JSON via the producer registry — this layer is free; need
to confirm at G33 land.)

bridge.

## § 6. Open issues for user to resolve pre-G33 land

(Analogous to G32's open-question section · all default suggestions retained
from the G32 candidate-A digest § "User 가 lock-in 시 답해야 할 open
question" item 1. Re-stated here for G33-land convenience.)

1. **Single subject vs multi-subject mean** for the Aura measured-oracle?
   - default: single subject `SC4001E0` · 5 eyes-closed Wake epochs · 30 s ·
     alpha-band integrated power.
   - alt: 3-subject mean (`SC4001E0` / `SC4002E0` / `SC4011E0`) — better
     statistical robustness but larger fetch + 3× scope_caveats prose.
2. **PASS criterion (a) vs (b)** — band-integrated alpha power (mean_rel_err
   ≤ 0.05) vs spectral-peak rel_err (≤ 0.02)?
   - default (G32 note): (a) band-integrated, mirror of solar 5% criterion.
3. **Welch averaging hexa-native vs bridge-trusted**?
   - default (G32 note): bridge-trusted (mirror of solar pvlib treatment).
   - If user prefers (b), this opens a κ-69 G33-β substrate round to port
     Welch averaging into hexa-native (next ARCH §11.4 Round 8 G-item
     candidate). Out of G33 land scope under default.
4. **PhysioNet fetch — bundled fixture vs live fetch**?
   - Open question NOT in G32 note. solar G29 path uses *live* MIDC fetch
     (network at run time). PhysioNet Sleep-EDF is multi-MB; live fetch is
     bandwidth + cache-policy concern; bundled fixture (commit a single
     30-s epoch in `exports/aura/fixtures/`) trades repo size for offline
     reproducibility.
   - **default recommendation**: live fetch with on-disk cache under
     `~/.cache/demiurge/physionet/sleep-edfx/` (mirror of how solar G29's
     MIDC CSV is treated — fetch + write to cache · re-fetch only if cache
     miss).
5. **Producer file naming** — `aura_physionet_eeg_verify.py` vs second
   `main()` in `aura_mne.py` vs `aura_psd_verify.py`?
   - default recommendation: **new file** `aura_physionet_eeg_verify.py`
     (parallel to `nrel_midc_pyranometer.py` naming · clear separation
     between analyze and verify producers · D72 ①b adapter pattern).
6. **Dispatcher wiring** — does the existing producer registry path auto-
   discover `aura + verify` JSON under `exports/aura/verify/<stamp>/`, or
   does G33 need an explicit `AuraVerifyProducer.swift` cockpit-side
   producer (mirror of `AuraAnalyzeProducer.swift`)?
   - investigation deferred to G33 land first step — check
     `ProducerRegistry` / `RecordLoader` for verify auto-discovery.
